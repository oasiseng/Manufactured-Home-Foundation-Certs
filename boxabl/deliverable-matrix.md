# Boxabl Deliverable Matrix

| Scenario | Typical deliverable | Notes |
| --- | --- | --- |
| Product package present, standard site, no custom accessory load path | Guidance memo or desk-review conclusion | Keep the deliverable aligned with the actual request. |
| Lender or processor only needs a limited engineering statement | Letter or scope clarification | Use only if the requested statement matches the defensible basis. |
| Custom platform, podium, or nonstandard support geometry | Stamped plan or design review path | Do not collapse this into a simple letter. |
| Stairs, decks, or landings create their own structural problem | Accessory-specific engineering scope | Separate the accessory issue from the base product when possible. |
| Field conditions conflict with submitted product basis | Site verification or revised package request | Resolve the discrepancy before closing the case. |
