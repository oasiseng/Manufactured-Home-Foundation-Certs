# Triage Decision Matrix

Use this matrix with the YAML rules when deciding whether a case can stay remote.

| Condition | Default output | Notes |
| --- | --- | --- |
| Full remote package with clear visibility and no red flags | `remote_review` | Proceed to technical review and template assembly. |
| Missing appraisal but otherwise strong visual package | `request_more_evidence` | Request either the appraisal or a lender-approved property summary. |
| Missing or inaccessible underside evidence | `site_tech_dispatch` | Conservative default because support and anchorage cannot be verified well. |
| Ambiguous foundation type or anchorage | `site_tech_dispatch` | Remote assumptions are not acceptable when the system itself is unclear. |
| Attached porch, deck, carport, or addition with unclear load path | `site_tech_dispatch` | Can stay remote only if the structure is well documented and clearly independent. |
| Visible drainage, slope, flood, or settlement concern | `site_tech_dispatch` | Requires in-person judgment unless the issue is clearly superficial. |
| File set is incomplete but the missing items are easy to obtain | `request_more_evidence` | Use the deficiency template and set a follow-up owner. |
| Lender or AHJ explicitly requires a site visit | `site_tech_dispatch` | Do not fight the instruction. |
| Boxabl case missing manufacturer package or module configuration | `request_more_evidence` | Boxabl reviews need product-specific documentation before a remote decision. |
